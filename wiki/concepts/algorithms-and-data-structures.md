# Algorithms and Data Structures

Algorithms and data structures form the core of efficient computation, balancing time and space complexity to solve problems.

## Core Concepts
- **Fundamental Structures**: Linked Lists, Stacks, Queues, Hash Tables, and Heaps [[uiuc-2023-cs225]].
- **Tree-based Structures**: BST, AVL Trees, B-Trees, and k-d Trees [[uiuc-2023-cs225]].
- **Graph Algorithms**: BFS, DFS, Dijkstra's Algorithm, and Minimum Spanning Trees [[uiuc-2023-cs225]].

## Hardware Considerations
Modern algorithm design must account for physical hardware characteristics beyond asymptotic complexity, including:
- Cache locality and memory hierarchies [[slotin-2026-algorithmica-hpc]].
- SIMD (Single Instruction, Multiple Data) parallelism [[slotin-2026-algorithmica-hpc]].
- Instruction-level parallelism and pipeline hazards [[slotin-2026-algorithmica-hpc]].

## Bias and Gaps
- **Theory vs. Practice**: Traditional academic courses often emphasize theoretical Big O complexity over actual hardware performance [[uiuc-2023-cs225]].
- **Portability vs. Performance**: Extreme optimization for specific hardware (SIMD, cache alignment) can reduce code portability [[slotin-2026-algorithmica-hpc]].
