Main thesis: The interviewer is evaluating *how* you think, not just if the code works.
- **Thinking Aloud**: Never be silent; explain the thought process continuously.
- **Iterative Refinement**: Start with la brute-force solution, then optimize.
- **Handling Blocks**: How to ask for hints and lead the interviewer to guide you.
- **Collaboration**: Treat the interview as a pair-programming session.
