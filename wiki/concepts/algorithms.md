# Algorithms

Algorithms are step-by-step procedures for calculations, data processing, and automated reasoning.

## Common Algorithm Paradigms
- **Brute Force**: Exhaustive search of all possibilities.
- **Divide and Conquer**: Breaking the problem into smaller sub-problems (e.g., Merge Sort, Quick Sort).
- **Dynamic Programming**: Solving sub-problems and storing results to avoid redundant work (e.g., Knapsack problem).
- **Greedy Algorithms**: Making the locally optimal choice at each step (e.g., Dijkstra's, Prim's).
- **Backtracking**: Exploring all paths and abandoning those that fail (e.g., N-Queens).

## Key Categories
- **Sorting**: Bubble, Insertion, Selection, Merge, Quick, Heap, Radix.
- **Searching**: Linear search, Binary search.
- **Graph Algorithms**: BFS, DFS, Dijkstra, A*, Bellman-Ford, Floyd-Warshall.
- **String Algorithms**: KMP, Boyer-Moore, Rabin-Karp.

## Analysis Metrics
- **Time Complexity**: Big O notation ($\Omega, \Theta, O$).
- **Space Complexity**: Auxiliary space required by the algorithm.
- **Stability**: Whether the algorithm preserves the relative order of equal elements (relevant for sorting).

## Counter-arguments
- Algorithmic efficiency is often secondary to maintainability and readability in industry code unless performance is critical.
