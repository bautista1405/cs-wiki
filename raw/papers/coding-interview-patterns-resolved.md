# Coding Interview Patterns
**Authors:** Alex Xu & Shaun Gunawardane
**Publisher:** ByteByteGo Inc. (2024)

## Summary
This document is a comprehensive guide to mastering coding interviews by focusing on 19 essential algorithmic patterns. Instead of memorizing individual problems, the book emphasizes recognizing underlying patterns that can be applied to various challenges. It covers 101 problems in total, ranging from basic data structures to complex dynamic programming and graph algorithms.

## Key Patterns and Chapters

### 1. Two Pointers
Used for searching pairs in sorted arrays or validating palindromes.
- Key problems: Pair Sum (Sorted), Triplet Sum, Is Palindrome Valid, Largest Container.

### 2. Hash Maps and Sets
Crucial for constant-time lookups and frequency counting.
- Key problems: Pair Sum (Unsorted), Verify Sudoku Board, Zero Striping.

### 3. Linked Lists
Focuses on pointer manipulation and structural changes.
- Key problems: Linked List Reversal, Remove Kth Last Node, Linked List Intersection, LRU Cache.

### 4. Fast and Slow Pointers
Commonly used for cycle detection and finding midpoints.
- Key problems: Linked List Loop, Linked List Midpoint, Happy Number.

### 5. Sliding Windows
Effective for problems involving contiguous subarrays or substrings.
- Key problems: Substring Anagrams, Longest Substring With Unique Characters, Longest Uniform Substring After Replacements.

### 6. Binary Search
Optimization for searching in sorted data.
- Key problems: Insertion Index, First and Last Occurrences, Cutting Wood, Rotated Sorted Array.

### 7. Stacks
Used for nested structures and maintaining order of operations.
- Key problems: Valid Parenthesis, Next Largest Number to the Right, Evaluate Expression.

### 8. Heaps
Ideal for tracking the Top-K elements or merging sorted lists.
- Key problems: K Most Frequent Strings, Combine Sorted Linked Lists, Median of an Integer Stream.

### 9. Intervals
Managing ranges and overlaps.
- Key problems: Merge Overlapping Intervals, Identify All Interval Overlaps, Largest Overlap.

### 10. Prefix Sums
Efficiently calculating sums of subarrays.
- Key problems: Sum Between Range, K-Sum Subarrays, Product Array Without Current Element.

### 11. Trees
Focuses on recursive traversals and hierarchical structures.
- Key problems: Invert Binary Tree, Balanced Tree Validation, Rightmost Nodes, Widest Level, BST Validation, LCA, Build Tree from Traversal, Max Path Sum.

### 12. Tries
Efficient for word searches and prefix matching.
- Key problems: Design a Trie, Search Words with Wildcards, Find All Words on a Board.

### 13. Graphs
Covers connectivity, traversal (BFS/DFS), and shortest paths.
- Key problems: Graph Deep Copy, Count Islands, Matrix Infection, Bipartite Validation, Longest Increasing Path, Shortest Transformation Sequence, Merging Communities.

### 14. Backtracking
Exploring all possible permutations and combinations.
- Key problems: Find All Permutations, Find All Subsets, N Queens.

### 15. Dynamic Programming (DP)
Optimizing problems with overlapping subproblems and optimal substructure.
- Key problems: Climbing Stairs, Minimum Coin Combination, Matrix Pathways, Neighborhood Burglary, Longest Common Subsequence, Longest Palindrome, Max Subarray Sum, 0/1 Knapsack.

### 16. Greedy Algorithms
Making the locally optimal choice at each step.
- Key problems: Jump to the End, Gas Stations, Candies.

### 17. Sort and Search
Combining sorting with searching techniques.
- Key problems: Sort Linked List, Sort Array, Kth Largest Integer.

### 18. Bit Manipulation
Low-level operations for optimization.
- Key problems: Hamming Weights of Integers, Lonely Integer, Swap Odd and Even Bits.

### 19. Math and Geometry
Handling coordinate planes, GCD, and overflow.
- Key problems: Spiral Traversal, Reverse 32-Bit Integer, Maximum Collinear Points.

## Core Findings
- **Pattern Recognition:** The ability to map a problem to a known pattern (e.g., using a "Sliding Window" for substring problems) is more valuable than solving a high volume of random problems.
- **Complexity Analysis:** Every solution is analyzed for Time and Space complexity (Big O), emphasizing efficiency.
- **Edge Case Handling:** A significant part of the resolution involves managing integer overflow (especially in 32-bit environments) and precision issues in geometry (using GCD for slopes instead of floats).
