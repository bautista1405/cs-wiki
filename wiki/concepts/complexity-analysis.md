    13|Complexity analysis is mandatory for every interview solution. It communicates the efficiency of the approach and justifies the choice of one algorithm over another.
    14|
    15|### Time Complexity
    16|- Measures how the runtime grows relative to the input size $n$.
    17|- Common complexities: $O(1), O(\log n), O(n), O(n \log n), O(n^2), O(2^n)$.
    18|
    19|### Space Complexity
    20|- Measures the extra memory used by the algorithm, excluding the input.
    21|- Critical for deciding between iterative and recursive approaches (Stack space).
    22|
    23|### Interview Expectation
    24|Candidates are expected to provide the complexity before writing code and discuss potential trade-offs (e.g., trading space for time via memoization).
