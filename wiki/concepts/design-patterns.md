# Design Patterns

Reusable solutions to commonly occurring problems in software design.

## Core Categories
- **Creational**: Object creation patterns (e.g., Singleton, Factory).
- **Structural**: Composition patterns (e.g., Adapter, Decorator).
- **Behavioral**: Communication patterns (e.g., Observer, Strategy).

## Sources
- [[refactoring-guru-2026-design-patterns]]: Comprehensive guide to classic design patterns.

## Bias/Context
- Patterns should be used as tools, not rigid rules. Over-applying patterns can lead to unnecessary complexity (patternitis).
